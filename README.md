# DHL-E-Commerce-Logistics-Database-SQL-Project-
Built a DHL e-commerce logistics SQL database with normalized tables and foreign keys; created analytical queries for shipment tracking, revenue analysis, and operational reporting.

**Project Overview**

This project demonstrates the design and implementation of a relational SQL database for a DHL-style e-commerce logistics system. The database tracks customers, shipments, packages, payments, employees, and branches, enabling operational and analytical reporting for logistics management.
The project focuses on database design, SQL querying, and business insights commonly required in real-world logistics and supply chain environments.

**Tools & Technologies**

Database: MySQL
Language: SQL
Concepts Used:
Relational Database Design,
Primary & Foreign Keys,
JOINS, Subqueries,
Aggregations (SUM, COUNT),
Data Filtering & Sorting.

**Database Schema
Tables Included:**

Customer – Stores sender and receiver details,
Package – Package weight, value, content, and delivery status,
Shipment – Links customers and packages with pickup & delivery details,
Payment – Shipment payment records and methods,
Employee – Employee roles and branch locations,
Branch – Branch details and manager assignments.

**Key SQL Queries & Analysis**

Total revenue generated from shipments,
Shipment delivery status (Delivered vs In Transit),
Top high-value packages,
Daily payment collection report,
Customer shipment activity (senders & receivers),
Employee and branch-level reporting,
Shipment tracking with package and payment details (JOINS).

**Insights**

Identified total revenue collected from shipment payments,
Tracked pending and delivered shipments using delivery dates,
Found top 2 highest-value packages,
Generated daily payment summaries for finance reporting,
Analyzed shipment volume per customer.
